Debugging & Timeouts

Als tests niet doen wat je verwacht

Wat gaan we leren?

Na dit blok kun je:

  • Cypress debugging tools effectief gebruiken
  • Timeouts begrijpen en configureren
  • Flaky tests oplossen

Debugging Tools

  • cy.log() - berichten in de Test Runner
  • .debug() - pauzeert en opent DevTools
  • cy.pause() - stopt de test
  • cy.screenshot() - maakt screenshot

Debugging in Actie

it('debug voorbeeld', () => {
  cy.visit('/products')

  cy.log('Zoeken naar producten...')

  cy.get('[data-cy="product-list"]')
    .debug()  // Opent DevTools, pauzeert hier
    .find('.product-card')
    .should('have.length.gt', 0)

  cy.pause()  // Test stopt hier, je kunt stappen
})

Timeouts Configureren

// Globaal in cypress.config.ts
defaultCommandTimeout: 10000,  // 10 sec default

// Per command
cy.get('[data-cy="slow-element"]', { timeout: 15000 })
  .should('be.visible')

// Per assertion
cy.get('[data-cy="loading"]')
  .should('not.exist', { timeout: 20000 })
